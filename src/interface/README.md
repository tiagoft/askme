# RTP Tree Visualizer

An interactive JavaScript-based visualizer for Recursive Topic Partitioning (RTP) trees. This tool allows you to visualize, explore, and export tree structures generated by the RTP algorithm.

## Files

- **tree_visualizer_standalone.html** - Main visualizer (recommended) - Works completely offline with no external dependencies
- **tree_visualizer.html** - Alternative version that uses CDN libraries (requires internet connection)
- **sample_tree.json** - Sample tree data for testing

## Features

### 1. 📊 Interactive Visualization
- **Tree Layout**: Hierarchical tree structure using D3.js
- **Node Interactions**: Click on any node to view detailed information
- **Zoom & Pan**: Navigate large trees with mouse wheel zoom and drag-to-pan
- **Color-Coded Branches**: 
  - Green links = YES branch (documents that match the question)
  - Red links = NO branch (documents that don't match the question)
  - Purple nodes = Internal nodes with questions
  - Blue nodes = Leaf nodes (no further splits)

### 2. 📋 View Questions
- Display the question used to split each node
- Identify leaf nodes (no split question)
- Navigate through the decision tree structure

### 3. 📈 Split Information & Metrics
View detailed metrics for each node:
- **Split Ratio**: Proportion of documents in each branch
- **NLI Calls**: Number of Natural Language Inference calls
- **Processing Time**: Time taken to process the split (ms)
- **Confidence**: Average NLI confidence scores
- And more...

### 4. 📄 Document Access
- View the count of documents in each node
- See document IDs for each node
- Track document distribution across the tree

### 5. 💾 Export Capabilities
- **Export to PDF**: Generate publication-ready PDF figures
- **Download SVG**: Get vector graphics for articles and presentations
- High-quality output suitable for academic papers and demos

## Usage

### Opening the Visualizer

Simply open the `tree_visualizer_standalone.html` file in your web browser (recommended):

```bash
# Using your default browser
open src/interface/tree_visualizer_standalone.html

# Or on Linux
xdg-open src/interface/tree_visualizer_standalone.html

# Or on Windows
start src/interface/tree_visualizer_standalone.html
```

Alternatively, you can serve it with a simple HTTP server:

```bash
# Python 3
cd src/interface
python -m http.server 8000

# Then visit: http://localhost:8000/tree_visualizer_standalone.html
```

**Note**: The standalone version works completely offline and has no external dependencies. If you prefer to use CDN libraries and have an internet connection, you can use `tree_visualizer.html` instead.

### Loading Tree Data

There are three ways to load tree data:

#### 1. Upload JSON File
1. Click "📂 Load from File"
2. Select your `.json` file containing the tree data
3. The tree will be visualized automatically

#### 2. Paste JSON Text
1. Paste your tree JSON into the text area
2. Click "📋 Load from Text"
3. The tree will be visualized automatically

#### 3. Load Sample Data
1. Click "🔬 Load Sample Data" to see an example tree
2. Great for testing and understanding the format

### Interacting with the Tree

- **View Node Details**: Click on any node (circle) to see:
  - Question used for splitting
  - Document count and IDs
  - Split metrics and statistics
  
- **Navigate**: 
  - Zoom: Use mouse wheel or trackpad pinch
  - Pan: Click and drag the background
  
- **Close Info Panel**: Click the "×" button or click outside the panel

### Exporting

Once a tree is loaded:

1. **Download SVG**:
   - Click "💾 Download SVG"
   - Get a vector graphic file
   - Perfect for high-quality figures in LaTeX documents and presentations

2. **Download PNG** (Standalone version only):
   - Click "🖼️ Download PNG"
   - Get a raster image file
   - Suitable for including in documents and presentations

3. **Export to PDF** (CDN version only):
   - Available in tree_visualizer.html
   - Click "📄 Export to PDF"
   - A PDF file will be downloaded automatically

## Tree JSON Format

The visualizer expects a JSON file with the following structure:

```json
{
  "documents": [0, 1, 2, 3, 4, 5],
  "question": "Is this about animals?",
  "metrics": {
    "split_ratio": 0.5,
    "nli_calls": 10,
    "total_time_ms": 1500.0,
    "medoid_nli_confidence_avg": 0.85,
    "llm_input_tokens": 0,
    "llm_output_tokens": 0,
    "faiss_search_time_ms": 0.0,
    "label_propagation_time_ms": 0.0,
    "llm_request_time": 0.0,
    "nli_time": 0.0,
    "num_nodes": 1
  },
  "left": {
    "documents": [0, 1, 2],
    "question": "Is this about cats?",
    "metrics": { ... },
    "left": null,
    "right": null
  },
  "right": {
    "documents": [3, 4, 5],
    "question": null,
    "metrics": null,
    "left": null,
    "right": null
  }
}
```

### Required Fields
- `documents`: Array of document indices (integers)
- `question`: String or null (null for leaf nodes)
- `left`: Left child node or null
- `right`: Right child node or null

### Optional Fields
- `metrics`: Object containing split metrics (can be null)

## Generating Tree JSON

You can generate tree JSON files using the RTPBuilder:

```python
from askme.rtp import RTPBuilder

# Initialize builder
builder = RTPBuilder(use_gpu=False)

# Build tree from text collection
text_collection = ["doc1", "doc2", "doc3", ...]
tree_root = builder(text_collection)

# Save to JSON
json_string = tree_root.model_dump_json(indent=2)
with open('my_tree.json', 'w') as f:
    f.write(json_string)
```

See `examples/use_rtp_builder.py` for a complete example.

## Troubleshooting

### Tree not loading?
- Check that your JSON is valid (use a JSON validator)
- Ensure the JSON follows the expected format
- Check browser console for error messages

### Export not working?
- Ensure you've loaded a tree first
- Check that your browser allows downloads
- For large trees, PDF generation may take a few seconds

### Tree looks cut off?
- Use zoom controls to zoom out
- The tree auto-centers on load
- Try resizing your browser window
